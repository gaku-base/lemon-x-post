# ポストコンシェルジュ v4.10.2 実装指示

対象リポジトリ:
- `gaku-base/lemon-x-post`
- 基準ブランチ: `main`
- 基準公開版: v4.10.1

## 目的

1. ヘッダーを現在の約半分の高さにする。
2. 左にサンドくん、右に次の2段ロゴを表示する。
   - 上段: `レモン apps`
   - 下段: `POST CONCIERGE`
3. 文字を明朝体から、すっきりしたサンセリフ体へ変更する。
4. ホーム画面追加時のアイコンを、現在ヘッダー内に埋め込まれている「サンドくん 執事版」画像から自動生成する。
5. 表示バージョンを `v4.10.2（2026-07-17）` にする。

## 変更ファイル

同梱の次の2ファイルで、リポジトリ上の同名ファイルを置き換える。

- `app-simple.js`
- `styles-simple.css`

または、`post-concierge-v4.10.2.patch` を適用する。

## 変更禁止

- Airリザーブ取得処理
- 予約状況判定
- 投稿文生成
- X文字数換算
- メニュー管理
- Cloudflare / GitHub Actionsの取得処理
- `index.html` 内のサンドくんBase64画像

## 実装上のポイント

- `app-simple.js` は、既存のサンドくん画像をCanvasへ描画して180/192/512pxのアイコンを生成する。
- iPhone向けに `apple-touch-icon` を動的設定する。
- PWA向けにmanifestを動的生成する。
- アイコン生成に失敗しても、アプリ本体の機能は止めない。
- ヘッダーはスマホ幅で約104px、370px以下で約96pxを目安とする。

## テスト

1. PC表示でロゴが1行に収まり、サンドくんが切れていない。
2. iPhone Safariでヘッダーが以前より低くなっている。
3. 「ホーム画面に追加」でサンドくんアイコンが候補になる。
4. かんたん／詳細の切替が動作する。
5. メニュー選択、予約状況取得、投稿文生成、コピー、X起動が従来どおり動作する。
6. コンソールに致命的エラーがない。
7. 表示バージョンが `v4.10.2（2026-07-17）`。
